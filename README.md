## 2. Назначение решения

Прототип «PSB Learn» — это веб-платформа для студента с личным кабинетом, списком курсов, заданиями и тестами, прогрессом, оценками и календарём дедлайнов.  
Решение демонстрирует:

- единое пространство для учебных активностей;
- личный кабинет студента с профилем и настройками;
- отображение текущих заданий, статуса выполнения и оценок;
- базовый чат по курсу;
- удобный запуск всего стенда локально через Docker.

Прототип рассчитан на демонстрацию функциональности и UX, работает локально без внешних зависимостей.

## 4. Структура проекта

Корень репозитория:

- `backend/` — backend-приложение (FastAPI, модели БД, миграции, настройки).
- `frontend/` — frontend-приложение (React + TypeScript, страницы и компоненты).
- `docker-compose.yml` — запуск PostgreSQL, backend и frontend одним командой.
- `README.md` — описание проекта и инструкция по запуску.

### 4.1. Backend

- `backend/app/main.py` — точка входа FastAPI.
- `backend/app/models/` — ORM-модели (пользователи, курсы, задания, прогресс, тесты и т.д.).
- `backend/app/schemas/` — Pydantic-схемы для API.
- `backend/app/api/v1/` — маршруты API (аутентификация, курсы, задания, оценки, профиль и др.).
- `backend/alembic/` — миграции базы данных.
- `backend/requirements.txt` — зависимости backend-части.

### 4.2. Frontend

- `frontend/src/pages/` — страницы интерфейса (логин, регистрация, дашборд, курсы, задания, профиль и др.).
- `frontend/src/components/` — общие компоненты (карточки курсов, кнопки, layout, хедер, сайдбар).
- `frontend/src/api/` — обёртки для вызова backend-API.
- `frontend/src/store/` — состояние приложения (authStore, uiStore).
- `frontend/src/theme/theme.ts` — тема и стили, в том числе градиенты ПСБ.

## 5. Быстрый запуск через Docker

### 5.1. Предварительные требования

- Docker Desktop (Windows/macOS) или Docker Engine (Linux).
- Свободные порты: **5173** (frontend), **8000** (backend), **5432** (PostgreSQL).

### 5.2. Клонирование репозитория

```bash
git clone https://github.com/NotForResell/psb-learn-super-red-rocket.git
cd psb-learn-super-red-rocket
5.3. Запуск
Из корня проекта:

bash
Копировать код
docker compose up --build -d
После успешного запуска:

frontend: http://localhost:5173

backend (Swagger UI): http://localhost:8000/docs

PostgreSQL: порт 5432 (параметры заданы в docker-compose.yml и используются backend-частью автоматически).

5.4. Демонстрационный вход
При инициализации базы создаётся демо-пользователь:

E-mail: student@example.com

Пароль: student123

Можно зарегистрировать нового пользователя через страницу регистрации.
Основные сценарии демонстрации:

вход под демо-аккаунтом;

просмотр списка курсов и заданий;

открытие курса и задания;

отправка решения и просмотр истории попыток;

просмотр вкладок «Прогресс», «Оценки», календаря;

работа с личным кабинетом (ФИО, аватар, смена пароля).

6. Локальный запуск frontend для разработки (опционально)
bash
Копировать код
cd frontend
npm install
npm run dev
По умолчанию dev-сервер поднимается на http://localhost:5173.
Backend в этом случае можно запускать отдельно из каталога backend (через uvicorn) или оставить вариант через Docker.

7. Известные ограничения прототипа
Файлы решений хранятся в упрощённом виде (для хакатона); интеграция с внешними хранилищами не реализована.

Ролевая модель ограничена, интерфейс ориентирован в первую очередь на студента.

Отсутствует полноценная админ-панель и масштабирование по пользователям и курсам — это задел для дальнейшего развития.
